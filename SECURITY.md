# Security Policy

> Responsible disclosure keeps everyone safe.

## Supported Versions

| Version / Branch | Supported |
|---|---|
| main (latest) | Yes |
| Older tagged releases | Best-effort only |

## Reporting a Vulnerability

Do not open a public GitHub issue for security vulnerabilities.

Report privately via:
1. GitHub Security Advisories for this repository
2. Email: anotherwaltzcompany@gmail.com (subject line: [SECURITY])

Include: description, reproduction steps, affected version/subproject, potential impact.

## Response Timeline

| Stage | Target |
|---|---|
| Acknowledge report | Within 72 hours |
| Initial assessment | Within 7 days |
| Fix & disclosure | Case-by-case |

## Scope

In scope: vulnerabilities in this repo's code (or any subproject folder for monorepos).
Out of scope: third-party dependency bugs (report upstream), physical access attacks, social engineering.

Thank you for reporting responsibly.
